Import a form

ProxForm reads two formats: a YAML-style indented file (human-friendly, easy to write in any text editor) or JSON (machine-friendly, what the Export button produces). Paste into the box below and click Create form, or scroll down for the syntax reference.

The two-minute version

A complete form definition looks like this:

title: Pre-appointment intake
description: Please fill this in before your visit.

# Patient Information
text*    half    Full name
date*    half    Date of birth
text     half    Phone
text     half    Email

# Reason for Visit
textarea*        Primary reason for visit
yesno            Have you been treated for this before?

Each non-blank line is either form metadata, a section header, a field, or an option attached to a field above. The whole file is saved locally in your browser — never uploaded anywhere.

Form metadata

Three keys, all optional:

title: Pre-appointment intake
description: Please fill this in before your visit.
  • title: — The form's name. Shows up in the patient's view and in your forms list.
  • description: — A one-line note shown under the title.
  • numbered: true — Auto-numbers questions (1., 2., …); sections aren't counted.

Section headers

A line starting with # creates a section band that groups the fields below it. An optional indented > line directly under the section attaches a short description.

# Patient Information
                 > Required so we can match you with your record.
# Medical History
# Consent & Signatures

Fields

The general field syntax is:

<type>[*] [half|third|quarter|full] <label>
  • <type> — one of the supported types (see below).
  • * — if present, the field is required.
  • half / third / quarter — optional column width. Consecutive same-width fields share a row.
  • <label> — the question text the patient sees.

Column-width examples

text*    half     First name
text*    half     Last name

number   third    Day
number   third    Month
number   third    Year

text     quarter  DD
text     quarter  MM
text     quarter  YYYY
text     quarter  CVV

The supported field types

text

Single-line text input.

text*    half    Full name
text             Allergies (none if blank)

textarea

Multi-line text input for longer answers.

textarea*        Describe your symptoms
textarea         Anything else we should know?

number

Numeric input (with the number keyboard on mobile).

number*  half    Weight (kg)
number   half    Height (cm)

date

Date picker.

date*    half    Date of birth
date             Date of last vaccination

radio

Single-choice from a list. Options go on indented - value lines.

radio*           Preferred contact method
                 - Email
                 - Phone
                 - SMS

checkbox

Multi-choice from a list. Same indented-options syntax as radio.

checkbox         Existing conditions (tick all that apply)
                 - Diabetes
                 - Hypertension
                 - Heart disease
                 - Asthma
                 - None of the above

yesno

Two-button Yes/No choice. No options needed.

yesno*           Are you currently pregnant?
yesno            Have you had this treatment before?

file

Patient attaches a file (camera-friendly on mobile). Base64-encoded and streamed peer-to-peer. Max 5 MB. In JSON set "accept" to restrict the picker.

file*    half    Photo of your ID
file             Insurance card

signature

Drawing pad for a handwritten signature. Finger, mouse, or stylus. Flattened to a base64 PNG and travels over the same chunked peer-to-peer path.

signature*       Patient signature
signature        Counter-signature (optional)

Help text under a field

A line starting with > right under a field (indented) attaches a one-line hint shown under the question on both the preview and the patient's view. The same syntax attaches a description to a section if it sits under a # header.

text*    half    Phone
                 > Best number to reach you during the day

number*          Weight (kg)
                 > Round to the nearest whole number

Default values

A line starting with = right under a field (indented) sets a default answer the patient sees pre-filled. The patient can still change it. Sections don't take defaults.

text             Country
                 = Belgium

radio*           Visit type
                 - First visit
                 - Follow-up
                 - Urgent
                 = Follow-up

yesno            Pregnant?
                 = no

date             Next appointment
                 = 2026-06-01

For checkbox fields, repeat the = line to pre-select multiple values:

checkbox         Allergies
                 - Pollen
                 - Latex
                 = Pollen

Validation rules

A line starting with ! right under a field (indented) attaches a validation rule. The patient's browser enforces it on submit — invalid values show the browser's native error bubble and block sending. One rule per line; repeat the prefix to stack rules.

number*          Weight (kg)
                 ! min=0
                 ! max=500

text*            Username
                 ! minlen=3
                 ! maxlen=20

text             Belgian VAT number
                 ! pattern=^BE\d{10}$
                 > Format: BE followed by 10 digits
  • min / max — numeric bounds (only on number).
  • minlen / maxlen — character-count bounds (on text / textarea).
  • pattern — JavaScript regex the whole value must match (only on text). No slashes; engine anchors implicitly.

Comments

Lines starting with // are ignored. Useful for notes or to temporarily disable a field while iterating.

// TODO: ask legal whether we need to collect insurance ID here
// text*    half    Insurance member ID

text*    half    Full name

What gets validated on import

  • Unknown field types raise an error — only the supported types are accepted.
  • Every radio and checkbox needs at least one option.
  • Pairs of consecutive half-width fields land on the same row; a lone half just takes the left column.

Multi-page forms

Splitting a long form keeps the patient focused. The patient view inserts Prev / Next buttons at each break and validates required fields per page before advancing. The builder previews the breaks as a striped divider but doesn't paginate so you can edit any field.

--- Step 2: Medical history

In JSON: { "type": "pagebreak", "label": "Step 2: Medical history" }. Label optional.

Conditional logic (show/hide)

A field can appear only when another field has a specific answer. Configure it in the builder. In JSON it's a per-field showIf object:

{ "id": "f2", "type": "textarea", "label": "Tell us more",
  "showIf": { "field": "f1", "op": "equals", "value": "Yes" } }

Operators: equals, notEquals, contains, notContains, empty, notEmpty. Hidden fields are skipped during required-field validation. YAML doesn't carry showIf yet.

Common mistakes

  • Option line not indented. Options for radio / checkbox must start with whitespace, then -, then the value.
  • Forgot the required star. * goes immediately after the type, no space.
  • Lone half. Only pairs with the next half-width field.
  • Unsupported types. Rich text and multi-file aren't in v1.

JSON format

If you prefer to generate forms programmatically or want a strict, machine-readable format, ProxForm also accepts JSON:

{
  "title": "Pre-appointment intake",
  "description": "Please fill this in before your visit.",
  "fields": [
    { "type": "section", "label": "Patient Information" },
    { "type": "text",     "label": "Full name",      "required": true, "column": "half" },
    { "type": "date",     "label": "Date of birth",  "required": true, "column": "half" },
    { "type": "text",     "label": "Phone",                              "column": "half", "hint": "Best number during the day" },
    { "type": "text",     "label": "Email",                              "column": "half" },
    { "type": "textarea", "label": "Primary reason for visit",          "required": true },
    { "type": "yesno",    "label": "Have you been treated for this before?" },
    {
      "type": "radio",
      "label": "How would you describe the pain?",
      "required": true,
      "options": ["None", "Mild", "Moderate", "Severe"]
    }
  ]
}

An exported form (with the Export button on the forms list) drops in directly — use it as a backup, share a template with a colleague, or generate forms from a script.

Private by design. ProxForm is GDPR-compliant by architecture — answers travel browser-to-browser, end-to-end encrypted. Like a sheet of paper, not a database.
